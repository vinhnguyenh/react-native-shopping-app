# Repository Instructions

## Scope

- These instructions apply to the entire repository.
- When working in `mobile/`, treat `mobile/.github/copilot-instructions.md` as the primary project guide.

## Mobile App

- For any task under `mobile/`, follow the rules in `mobile/.github/copilot-instructions.md`.
- Use the matching guidance in `mobile/.github/agents/` based on task type:
  - `coding.agent.md` for implementation and refactoring
  - `planning.agent.md` for feature breakdown and execution plans
  - `review.agent.md` for code review, bug/risk analysis, and architecture audits
  - `testing.agent.md` for unit, integration, and E2E testing work
- Apply the standards in `mobile/.github/instructions/` for all relevant React Native and TypeScript files.

## Backend

- No backend-specific instruction set is defined here yet.
- If backend-specific rules are added later, prefer adding them to this file as an explicit `backend/` section.
